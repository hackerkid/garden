Encoding

tags: #public

  • rolling upgrades - Deploy the new version to a few nodes at time. Make sure that the new version is running smoothly for all.
  • backward compatibility - New code can read data written by the old code.
  • forward compatiblity - Old code can read data written by new code.
  • Programs normally work with data in 2 representations
    • memory - Data is kept in structs, arrays, lists, hash table etc.
      • Optimized for efficent access by the CPU normally using a pointer.
    • sequence of bytes - When you want to send the data over the network or store in a file.
  • Translation from in memory representation to seqeuence of byes is called encoding. Also called as serializing, marshaling.
  • The reverse is called as decoding, parsing, deserialization, unmarshaling etc.
  • Many languages support encoding/decoding.
    • eg Pickle of Python, Marshal of Ruby
    • They are convinent cause you need to write minimal code.
    • But there are a lot of issues
      • Highly dependent on the language.
      • Decoding involves creating new classes which can be used by an attacker to execute random code.
      • Versioning of data is an afterthought.
      • Encoding and decoding is not very efficent.
  • Alternative is to use standard encodings like JSON, XML etc that are supported by many programming languages.
