docker-compose

tags: #public #docker

  • Automatically builds (or get images from a docker hub) and start the containers (services)
  • docker-compose.yaml is used for specifying how to start each container
    • A name to each container (service)
    • Dockerfile to build or which image to get
    • How to map the port requests from the host machine to the container
    • How to mount volumes etc
  • Docker compose also starts a virtual network.
    • So you can create requests to http://database, http://web, http://product-service etc
  • You can also start docker-compose in detached mode.
version : '3'
services:
	recommendtion-service:
		build: ./recommendation
		volumes:
			- ./recommendation:/home/recommendation
		ports:
			- 8001:80
	newsfeed-service:
		image: django:python-3
		volumes:
			- ./newsfeed:/home/newsfeed
		ports:
			- 8002:80
	depends_on:
		- recommendation-service
